Subnavbar Usage

The hc-subnav component is an optional part of the page header, it is not required. It is meant to be a flexible area that can be used as per the needs of the application. Some may choose to use it for breadcrumbs, others for secondary navigation, filtering, search, etc. hc-subnav with fill the width of the container it is placed in. It may be used either in conjunction with hc-navbar as described below, or independently as seen on the header of this demo page. If used independently, a fixedTop parameter may be set to pin the subnav to the top of the page below the primary nav.

Angular Component

        
<hc-subnav>
    <div>Content goes here</div>
    <div hcSubnavRight>
        <button hc-button color="primary">Example</button>
    </div>
</hc-subnav>

hc-subnav may be placed directly on the page or included within the hc-navbar tag to attach it to the primary navbar. 

        
<hc-navbar>
    <hc-app-switcher></hc-app-switcher>
    <hc-navbar-link></hc-navbar-link>
    <hc-navbar-link></hc-navbar-link>
    <hc-navbar-icon></hc-navbar-icon>
    <hc-subnav></hc-subnav>
    <hc-navbar-menu></hc-navbar-menu>
</hc-navbar>

Directives

hcSubnavRight
Can be added to a div or individual component to align it to the right side of the subnav. Primary action buttons are often placed on the right side of the subnav.

Component Parameters

fixedTop Type: Boolean Statically positions the subnav below the navbar (not needed subnav is included in hc-navbar)
