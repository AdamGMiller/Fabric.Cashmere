Color Variations
Above are all acceptable colors and their active states. Green should be used for most cases and especially cases the action is positve or in agreement. For negative actions use red. The purple and gray variations should be used only in cases where more colors are needed or if they are being placed on backgrounds which green doesn’t contrast well enough. Icons, if needed, should match text color of the button.


Usage
Generally, there should only be one primary button per page. All other interactive items should either be secondary buttons or text links. A few scenarios in which there would be multiple primary buttons: a modal with both an “accept” and “decline” action, a page who’s content are non related and each content section have disparate actions associated with them. For modals the preference is one primary button and a cancel action set as a text link. Below are some examples of buttons usage.



Angular Component

HTML TypeScript 
<hc-button
	[options]="{
		buttonName: 'Save',
		buttonType: 'primary',
		buttonDisplayed: true
	}">
</hc-button>

HTML TypeScript 
import { ButtonModule } from '@healthcatalyst/cashmere';

@NgModule({
  imports: [
     ...,
    ButtonModule,
  ],
  declarations: [...],
  exports: [..]
})
export class SomeModule { }



Component Parameters

buttonName Type: String The text that will appear on the button
buttonIcon Type: String The font-awesome icon that will appear on the button ('fa fa-plus')
buttonType Type: 'primary-green',
'primary-red',
'primary-purple',
'primary-gray',
'secondary',
'tertiary'		 Button theme (see examples above)
buttonDisabled Type: Boolean Boolean value that enables/disables the button
buttonDisplayed Type: Boolean Boolean that will hide/show the button
tooltip Type: String Text value that will appear on button hover
